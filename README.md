# DL Dimensional Metrology

Deep Learning pipeline for automated visual inspection of digitized industrial components, combining a custom Convolutional Neural Network (CNN) for binary conformance classification and a YOLOv8 architecture for precise spatial defect localization.

The dataset consists of color-deviation maps — images generated by overlaying digitized 3D point-clouds onto their CAD reference models — produced by industrial-grade scanners (HandScan, MetraSCAN). Each image encodes geometric deviation as a thermal color scale: warm colors indicate excess material, cold colors indicate material absence.

---

## Background

Industrial dimensional metrology has evolved significantly with the adoption of 3D digitization systems. Modern scanners capture complex geometries rapidly and non-destructively, generating high-resolution point-cloud representations of manufactured parts. However, interpreting the resulting deviation maps — comparing scanned geometry against CAD references — still depends heavily on human analysts, who visually evaluate thousands of inspections per production cycle.

This reliance on manual interpretation introduces a well-documented bottleneck: human fatigue, varying experience levels, and subjective judgment create inconsistency at scale. The problem compounds when dimensional tolerances are tight and production volumes are high.

This project was developed alongside academic research on the application of deep learning to dimensional metrology. The core idea: train convolutional models directly on the color deviation maps that metrologists already produce, automating both the binary conformance decision and the spatial localization of non-conforming regions.

This is an applied research project — not a production tool. The main limitations are documented below.

## Dataset

The images used in this project were generated from proprietary industrial inspection data. Due to confidentiality agreements with the manufacturing partner, **the dataset is not included in this repository**.

### Data Generation Process

Each sample originates from a standardized metrology workflow:

1. **3D Digitization** — Parts are scanned using optical devices (HandScan or MetraSCAN), producing `.stl` or point-cloud files.
2. **CAD Alignment** — The scanned geometry is aligned to the nominal CAD model using reference datum points (geometric zeros defined in the engineering drawing), following ASME Y14.5 or applicable ISO norms.
3. **Color Deviation Map** — The software overlays the scan onto the CAD and renders a color map encoding surface deviation. Dimensional tolerances (often derived from ISO 8062-3 or customer-specific tables) define the acceptance band.
4. **Labeling** — Images are classified as *conforming* or *non-conforming* based on the applied tolerance thresholds. For YOLO training, the specific defective regions are annotated manually using bounding boxes.

### Dataset Composition

To expand the dataset volume beyond what single-part production batches provide, geometric variation was simulated: small rotations and translations were applied to existing scan alignments from their datum reference points, reproducing realistic manufacturing variability. This approach generated multiple dimensional variants of the same component, significantly increasing training sample diversity.

| Configuration | Samples | CNN Accuracy |
|---|---|---|
| Initial validation set | 46 | 1.00 (memorized — not generalizable) |
| Intermediate set | 215 | 0.89 |
| Full training set | 827 | **0.97** |

## Scripts

| Script | Purpose |
|---|---|
| `scripts/classify_images.py` | OpenCV-based interactive viewer for rapid manual sorting of raw scan images into `good/` and `bad/` folders. Supports undo, progress tracking and keyboard shortcuts. |
| `scripts/yolo_annotation.py` | Fullscreen annotation tool for drawing bounding boxes over defective regions in non-conforming images. Exports coordinates in normalized YOLO format. |
| `scripts/yolo_dataset_split.py` | Applies stratified 70/20/10 train/val/test splitting independently per class, then assembles the final `dataset_yolo/` tree compatible with Ultralytics. |
| `scripts/data.yaml` | YOLO dataset configuration file. Defines relative dataset paths, number of classes (`nc: 1`) and class name (`defect`). Update paths before training. |

Every script uses relative paths and requires no hardcoded machine-specific configuration.

## Results Summary

### CNN Binary Classifier

Trained on 827 color deviation map images across two classes (conforming / non-conforming):

- **Accuracy**: 97%
- **Precision**: 97%
- **Recall**: 97%

The model was evaluated on an independent held-out test set (30% of the dataset), with stratified splitting preserving class proportions. The strong performance confirms that convolutional filters trained on thermal deviation maps successfully capture the visual signature of dimensional non-conformance.

Data augmentation (rotations, zoom, brightness shifts, shear) was tested but produced no measurable accuracy improvement — likely because the color gradient patterns encoding conformance are sensitive to geometric distortion. The final model was trained without augmentation.

### YOLOv8 Spatial Locator

Fine-tuned from YOLOv8n (nano) COCO pre-trained weights over 150 epochs on the annotated defective regions:

| Metric | Value |
|---|---|
| Precision | 69.7% |
| Recall | 45.5% |
| IoU (> 0.3 threshold) | 71.7% |
| Classification Accuracy | 98% |

→ **Strong spatial awareness, constrained recall due to dataset size**

The 98% classification accuracy confirms the model reliably distinguishes images containing defects from clean ones. The 45.5% recall reflects the inherent difficulty of localizing small, ambiguous boundary regions in manually annotated industrial data, where inter-annotator variability affects the ground-truth labels themselves. IoU above 71% indicates that detected regions overlap substantially with annotator-defined zones of interest.

## Key Design Choices (and Why)

**Why color deviation maps and not raw point clouds?**
Point-cloud data requires specialized volumetric preprocessing pipelines. Deviation maps are already the standard output of industrial metrology software (e.g. PolyWorks, GOM Inspect), making this approach directly compatible with existing production workflows. No additional 3D preprocessing is needed — the images feeding the CNN are identical to what metrologists already review manually.

**Why 128×128 pixel input resolution?**
After inspecting the deviation maps, the defect-discriminating features (color bands, hot/cold transitions) are coarse geometric patterns, not fine-pixel details. 128×128 provides sufficient spatial resolution to capture these transitions while allowing large batch sizes and fast training on standard hardware.

**Why 3 convolutional blocks with 32→64→128 filters?**
The progressive depth increase matches the hierarchical nature of the task. Early filters capture edges and color gradients. Mid-level filters learn thermal band shapes. Deep filters encode the conformance/deviation patterns that distinguish classes. MaxPooling 2×2 after each block reduces spatial dimensionality while preserving the most prominent activations.

**Why Dropout(0.5) after the Dense layer?**
With fewer than 1,000 samples, the network is at high risk of memorizing training examples. Dropout forces the model to learn redundant, distributed representations — increasing generalization without requiring additional labeled data.

**Why EarlyStopping and ReduceLROnPlateau?**
With 100 epochs configured, both callbacks prevent wasted compute and degradation. EarlyStopping halts training if validation loss does not improve for 10 consecutive epochs. ReduceLROnPlateau halves the learning rate when validation stalls, allowing the optimizer to fine-tune around local minima rather than oscillating.

**Why data augmentation was dropped?**
Augmentation was tested with rotations, zoom, shear, horizontal flips and brightness adjustments. None produced accuracy gains. The hypothesis: color-gradient patterns in deviation maps encode directional physical information (excess vs. absence of material). Rotating or flipping these maps changes the semantic meaning of the color distribution, introducing noise rather than useful variance.

**Why YOLOv8n (nano) and not a larger variant?**
The dataset contains fewer than 300 labeled defective regions — insufficient to justify the parameter count of YOLOv8m or YOLOv8l. The nano variant (≈3.2M parameters) trains in reasonable time, avoids severe overfitting, and still benefits fully from COCO transfer learning. Scaling up is straightforward once more annotated data becomes available.

**Why transfer learning from COCO?**
COCO pre-training provides convolutional filters already capable of detecting edges, textures, and object boundaries across diverse visual conditions. Fine-tuning from these weights — rather than training from scratch — significantly accelerates convergence and improves spatial localization even on a domain-specific dataset of industrial images.

## How to Reproduce

To apply this framework to a new dataset of color deviation maps:

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Place raw deviation map images in data/raw/

# 3. Sort images interactively into conforming / non-conforming
python scripts/classify_images.py
# Controls: [g] = good, [b] = bad, [SPACE] = undo, [q] = quit

# 4. (YOLO only) Annotate bounding boxes on non-conforming images
python scripts/yolo_annotation.py
# Controls: draw with mouse, [s] = save & next, [z] = undo last box, [q] = quit

# 5. (YOLO only) Split dataset into train/val/test structure
python scripts/yolo_dataset_split.py

# 6. Run the CNN training notebook
#    Open notebooks/01_CNN_Binary_Classification.ipynb

# 7. Run the YOLO fine-tuning notebook
#    Open notebooks/02_YOLO_Defect_Localization.ipynb
```

## Limitations

- **Proprietary data dependency**: The trained models are not included. The framework is architecture-ready but requires a new labeled dataset of deviation maps to be trained from scratch for a different part family or manufacturing context.
- **Annotation subjectivity**: Bounding box labels for the YOLO model were drawn manually. Regions of gradual deviation (near the tolerance boundary) introduce inter-annotator variability that directly affects recall and IoU metrics.
- **Single defect class**: The current YOLO configuration detects a single generic `defect` class. Distinguishing between defect types (excess material, material absence, surface roughness anomalies) would require re-labeling with multiple classes and a larger dataset.
- **Static tolerance thresholds**: Conformance labels are derived from fixed tolerance tables. Parts accepted under one customer specification may be non-conforming under another. The model learns the labeling convention used during training, not an absolute physical truth.
